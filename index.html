Mach-O初探

发表于 2018-03-09

Mach-O 是 Apple 系统上（包括 MacOS 以及 iOS）的可执行文件格式，一个Mach-O文件包含：文件头Header、加载命令Load commands、具体数据Segment&Section。

官方文档中的结构：

使用MachOView打开结构如下：

Header

Header中包含Mach-O的基本信息，它在文件中的作用主要是：使系统能够快速定位其运行环境以及文件类型等等。Header结构：

  • magic：魔数，用于快速确认该文件用于64位还是32位
  • cputype：CPU类型，比如 arm
  • cpusubtype：对应的具体类型，比如arm64、armv7
  • filetype：文件类型，比如可执行文件、库文件、Dsym文件，demo中是2 MH_EXECUTE，代表可执行文件
  • ncmds ：加载命令条数
  • sizeofcmds：所有加载命令的大小
  • reserved：64位的保留位
  • flags：dyld加载标志位

Load commands

Load Commands 是跟在 Header 后面的加载命令区，所有 commands 的大小总和即为 Header->sizeofcmds 字段，共有 Header->ncmds 条加载命令。

Segment&Section

加载数据时，主要加载的就是LC_SEGMET或LC_SEGMENT_64。

使用命令行操作iOS模拟器

发表于 2017-12-25

查看当前启动的模拟器

xcrun simctl list devices | grep Booted

输出结果：

iPhone 7 (5F0B29B7-4243-41F6-BB57-132ACD9B7A2A) (Booted)

查看某个模拟器内的应用

这个命令会输出包含系统应用在内的所有应用的信息，比如 应用名、BundleID、沙盒路径等等。

查看单独某个应用的信息

xcrun simctl appinfo booted BUNDLEID

xcrun simctl appinfo booted com.apple.Maps

这里 booted 的意思是说只在启动的模拟器中查找，也可以填写模拟器的 UUID。

快速找到 app 的沙盒文件夹

有一个关于simctl的命令。你可以获取到 app 的沙盒文件系统通过一条简单的命令。你只需要知道 app 的 bundle identifier 并执行以下命令：

xcrun simctl get_app_container booted
